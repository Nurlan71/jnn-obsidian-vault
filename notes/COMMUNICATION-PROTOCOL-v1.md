# 📋 JNN Communication Protocol v1.0
> Обновлено: 01.06.2026 | Статус: АКТИВЕН

## 🎯 Цель
Единая система коммуникации между всеми агентами JNN и сотрудниками. Все важные события фиксируются → ничего не теряется.

## 📁 Структура

```
/opt/shared/
├── journal/                          # ЕДИНЫЙ ЖУРНАЛ
│   ├── daily_journal.json            # Все события (агрегатор)
│   └── events/                       # Отдельные файлы событий
│       ├── 2026-06-01.jsonl
│       └── 2026-06-02.jsonl
├── messages/                         # МЕЖАГЕНТНЫЕ СООБЩЕНИЯ
│   ├── inbox/                        # Входящие для агентов
│   │   ├── marketer_inbox.json
│   │   ├── rop_inbox.json
│   │   ├── sales_inbox.json
│   │   ├── production_inbox.json
│   │   ├── finance_inbox.json
│   │   ├── hr_inbox.json
│   │   └── navigator_inbox.json
│   └── outbox/                       # Исходящие от агентов
│       ├── from_marketer/
│       ├── from_rop/
│       ├── from_sales/
│       └── from_production/
├── handoff/                          # ПЕРЕДАЧА СДЕЛОК МЕЖДУ ОТДЕЛАМИ
│   ├── marketing_to_sales/           # Маркетолог → Продажник
│   ├── sales_to_production/          # Продажник → Производство
│   └── production_to_sales/          # Производство → Продажник (готово)
└── reports/
    └── daily_digest-YYYY-MM-DD.md    # Ежедневная сводка
```

## 📝 Формат записи в журнал (Event)

```json
{
  "id": "evt-xxxxx",
  "timestamp": "YYYY-MM-DD HH:MM TZ",
  "agent": "marketer | rop | sales | production | finance | hr | navigator | human",
  "author": "Имя агента или сотрудника",
  "category": "lead | order | task | problem | handoff | decision | info",
  "title": "Краткое описание (1 строка)",
  "details": "Подробности",
  "related_to": ["evt-yyyyy", "evt-zzz"],
  "status": "new | in_progress | done | cancelled",
  "notify": ["rop", "navigator"]
}
```

## 🔄 Правила записи

### ОБЯЗАТЕЛЬНО записывать:

| Событие | Кто записывает | Кого уведомить |
|---------|---------------|----------------|
| Новый лид/клиент | Маркетолог | РОП, Навигатор |
| Лид → Активный клиент | Продажник | РОП |
| Новый заказ | Продажник | Производство, Финансист |
| Заказ в работе | Производство | Продажник |
| Заказ готов | Производство | Продажник, Финансист |
| Получена оплата | Финансист | Навигатор |
| Проблема/блокировка | Любой | Навигатор, РОП |
| Важное решение | Любой | Навигатор, Нурлан |
| Передача между отделами | Оба | Все заинтересованные |

### Как записывать:
1. **Первым делом** — запиши событие в журнал
2. Затем — выполни действие
3. Обнови статус когда закончишь

## 🔀 Протокол передачи между отделами (Handoff)

### Маркетолог → Продажник
```
Маркетолог создал лид → Записал в журнал (category=handoff)
                    → Положил файл в /handoff/marketing_to_sales/
                    → Уведомил Продажника (inbox)
Продажник получил → Прочитал в inbox
                  → Принял / Отклонил
                  → Обновил статус в журнале
```

### Продажник → Производство
```
Продажник получил заказ → Записал в журнал (category=order)
                        → Передал в /handoff/sales_to_production/
                        → Уведомил Производство (inbox)
Производство получило → Прочитало в inbox
                      → Подтвердило приём
                      → Обновило статус
```

### Производство → Продажник (готовность)
```
Производство завершило → Записало в журнал
                       → Уведомило Продажника и Финансиста
                       → Обновило статус заказа
```

## 👥 Сотрудники (люди)

### Как сотрудники взаимодействуют:
- **Тимур** (маркетолог) → пишет @JNN_Marketolog_bot → тот записывает в журнал
- **Руслан** (продажник) → пишет @JNN_Sales_bot → тот записывает в журнал  
- **Айгерим** (РОП) → пишет @JNN_ROP_bot → тот записывает в журнал
- **Алина** (финансы) → пишет @JNN_FinManager_bot → тот записывает в журнал

### Правило "Записал → Сделал"
1. Сотрудник пишет боту о событии
2. Бот СПРАШИВАЕТ: "Записать в журнал?"
3. Сотрудник подтверждает → бот записывает
4. Бот уведомляет заинтересованных

## ⏰ Автоматические проверки (Cron)

| Время (BJT) | Задача | Описание |
|-------------|--------|----------|
| 08:00 | Утренний дайджест | Все новые события за ночь → сводка |
| 14:00 | Проверка статусов | Не завис ли кто-то? Просрочки? |
| 20:00 | Вечерний дайджест | Все события дня → Нурлану |
| Каждый час | Мониторинг inbox | Есть ли непрочитанные сообщения? |

## 📊 Метрики коммуникации

Каждую неделю Navigator анализирует:
- Сколько событий записано
- Сколько передач между отделами
- Среднее время реакции на событие
- Сколько событий без записи (потеряно)

## 🔐 Права доступа

| Агент | Запись | Чтение | Уведомление |
|-------|--------|--------|-------------|
| Navigator | ✅ Все | ✅ Все | ✅ Все |

| РОП | ✅ Свои | ✅ Продажи, лиды | → Продажник, Навигатор |

| Производство | ✅ Свои | ✅ Заказы | → Продажник, Финансист |
| Финансист | ✅ Свои | ✅ Финансы | → Навигатор, Нурлан |
| HR | ✅ Свои | ✅ Кадры | → Навигатор |
