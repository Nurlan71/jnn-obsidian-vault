## Жёсткие правила работы
1. Перед любой настройкой — сначала проверять память и текущее состояние
2. Не повторять то, что уже сделано
3. Доверять предыдущим результатам, если нет явной ошибки
4. Проверять реальный результат, не верить "exit code 0"
5. Экономия токенов: один полезный шаг лучше десяти проверок
6. При ошибке — спросить Нурлана, не идти в настройки

### Правило управления инфраструктурой (HITL)
1. **Запрет на «грязную» модификацию** SOUL.md/config.yaml
2. **Делегирование технической роли:** субагент-разработчик (jnn-developer)
3. **Обязательный аппрув:** diff → одобрение Нурлана → выполнение
4. **Экономия контекста:** отвечать строго по вопросу
